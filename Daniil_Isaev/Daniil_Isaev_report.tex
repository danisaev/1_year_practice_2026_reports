\section{Постановка задачи}

Работа выполняется в проекте UCFS. \textbf{Целью работы является} создание простых примеров использования инструмента, включая реализацию кода и интеграцию инструкции по запуску в основную документацию.

Результаты работы необходимы для новых пользователей инструмента, которым требуется наглядная демонстрация его возможностей и понятное руководство для быстрого старта. В выполнении работы заинтересованы сопровождающие проекта UCFS, поскольку это должно снизить порог входа для новых контрибьюторов и пользователей.

\section{Описание предлагаемого решения}

UCFS представляет собой инструмент для работы с задачами на пересечении контекстно-свободных языков и ориентированных графов с метками на рёбрах, основанный на алгоритме GLL. До выполнения данной работы в проекте уже существовали примеры использования, однако они были сложными и громоздкими, что затрудняло начальное освоение инструмента. Аналоги из других проектов часто страдают той же проблемой: отсутствие простых, наглядных сценариев повышает порог входа для новых пользователей.

Предлагаемое решение заключается в создании набора минимальных и понятных примеров, которые позволяют последовательно познакомиться с возможностями UCFS. В отличие от существовавших ранее сложных примеров, разработанные сценарии намеренно упрощены: графы невелики по размеру, а их структура интуитивно понятна. Ключевым примером выбрана грамматика $a^nb^n$ --- она не имеет принципиального значения для демонстрации возможностей инструмента, но была взята из учебного пособия\footnote{Ссылка на пособие: \url{https://github.com/FormalLanguageConstrainedPathQuerying/FormalLanguageConstrainedReachability-LectureNotes} (дата обращения: 11.03.2026).}, что обеспечивает методическую обоснованность выбора. Для описания грамматики использовался встроенный DSL проекта UCFS, а графы представлены в формате \texttt{.dot}.

Принятые решения обоснованы следующим образом: простота примеров снижает порог входа и подготавливает новых пользователей к работе со сложными графами, имеющими практическое применение. Интеграция инструкции по запуску непосредственно в основной файл README (вместе с визуализациями графов в формате SVG) устраняет необходимость поиска отдельной документации.

Реализация решения осуществлялась с использованием языка Kotlin (код примеров), DSL для задания грамматики, формата \texttt{.dot} для описания графов, консольной утилиты dot (Graphviz) для конвертации графов в SVG и Markdown для оформления документации.

\section{Апробация}

Проверка разработанных примеров осуществлялась следующим образом. Каждый граф был вручную исследован на корректность структуры, после чего с помощью инструмента UCFS для него был получен SPPF (Shared Packed Parse Forest). Ручной разбор полученного SPPF дал результаты, идентичные результатам исходного поиска по графу, что подтверждает корректность работы примеров.

Запуск примеров выполняется единой командой через консольный сценарий. Несмотря на отсутствие формальных отзывов от сторонних пользователей, проблем с переносимостью и запуском на других устройствах ожидаться не должно благодаря стандартизированному формату графов (\texttt{.dot}) и отсутствию внешних зависимостей, специфичных для окружения разработки.

Оценка успешности работы проводилась по следующим критериям: все примеры успешно запускаются, результаты их работы совпадают с ожидаемыми, а обновлённый файл README стал более понятным для начинающих пользователей. 

На момент написания отчёта пуллреквест находится на рассмотрении (пометка об исправлении статуса: при включении в основную ветку следует указать, что решение принято и интегрировано). Предполагается, что разработанные примеры будут включены в основную ветку проекта UCFS в качестве учебных материалов для новых пользователей.

Материалы работы доступны в пуллреквесте: \url{https://github.com/vkutuev/UCFS/pull/3}.
